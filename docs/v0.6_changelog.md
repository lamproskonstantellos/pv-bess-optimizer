# PV+BESS optimizer — v0.6 changelog

This file is the contract for the v0.6 feature pack. It enumerates every
file touched, every public-API signature change, every workbook key
added/renamed/removed, every breaking change to run outputs, and the
five-line "what changed" diff a v0.5 user will see.

The phases below are sequenced exactly as implemented:

* **Phase 1** — workbook schema restructuring (breaking).
* **Phase 2** — Year-0 CAPEX / Year-1 operations convention.
* **Phase 3** — PV-only / BESS-only / hybrid asset support.
* **Phase 4** — configurable rolling-horizon uncertainty (4-source comparison).
* **Phase 5** — plot scope unification (daily included).
* **Phase 6** — merchant-mode plots (no load).
* **Phase 7** — new financial KPIs and plots (LCOE, LCOS, capacity factor, lifetime cycles).
* **Phase 8** — documentation + README badges.
* **Phase 9** — test pass.
* **Phase 10** — v0.5 leftover audit.

---

## 1. Files touched by phase

## 3. Workbook keys

### Added

* **`project` sheet → `# system_sizing`**: same keys as v0.5 `# system`
  minus the BESS-internal ones — `pv_nameplate_kwp`, `bess_power_kw`,
  `bess_capacity_kwh`, `battery_hours`, `p_charge_max_kw`,
  `p_dis_max_kw`, `p_grid_export_max_kw`.
* **`project` sheet → `# bess_operation`** (new): `efficiency_charge`,
  `efficiency_discharge`, `soc_min_frac`, `soc_max_frac`,
  `initial_soc_frac`, `terminal_soc_equal`, `max_cycles_per_day`.
* **`economic` sheet → `# uncertainty`** (new, 11 keys):
  `uncertainty_enabled`, `uncertainty_compare_sources`,
  `uncertainty_n_seeds`, `uncertainty_window_hours`,
  `uncertainty_commit_hours`, `uncertainty_dam_enabled`,
  `uncertainty_pv_enabled`, `uncertainty_load_enabled`,
  `uncertainty_sigma_dam`, `uncertainty_sigma_pv`,
  `uncertainty_sigma_load`.

### Renamed / vocabulary widened

* `plot_daily_year1` (bool) → `plot_daily_scope` (`none` | `year1_only`
  | `all`). Default `year1_only`.
* `plot_yearly_scope` accepted values widen from `none` | `all` to
  `none` | `year1_only` | `all`.

### Removed

* **`project` sheet → `# system`** (group label only — the keys move to
  `# system_sizing` and `# bess_operation`).
* **`project` sheet → `# optimization`** (entire group): four keys
  removed from the workbook —
    * `weight_curtail_tiebreak` → moved to
      `pvbess_opt.optimization._WEIGHT_CURTAIL_TIEBREAK_EUR_PER_KWH`.
    * `weight_cycles_term` → moved to
      `pvbess_opt.optimization._WEIGHT_CYCLES_TERM_EUR_PER_MWH`.
    * `solver_mip_gap` → CLI-only via `--mip-gap` (already exposed).
    * `solver_time_limit_seconds` → CLI-only via `--time-limit`
      (already exposed).

### Semantics changes

* `pv_nameplate_kwp = 0` no longer means "infer from timeseries". It
  means "PV is not part of the project" — the optimizer pins all PV
  flows to zero.
* `bess_power_kw = 0` no longer falls back to `p_dis_max_kw`. It means
  "BESS is not part of the project" — the optimizer pins all BESS
  variables to zero.
* `pv_nameplate_kwp = 0 AND bess_power_kw = 0` now raises
  `ValueError("Both pv_nameplate_kwp and bess_power_kw are zero …")`.
* `project_start_year` is now exclusively the calendar year of **Year
  1** (first operating year). Year 0 (CAPEX) is `project_start_year - 1`.

### Loader behaviour for legacy v0.5 workbooks

* Keys from the dropped `# optimization` group: a single WARNING listing
  every removed key found, then the keys are ignored.
* `plot_daily_year1`: a WARNING saying "renamed to `plot_daily_scope`;
  use 'none' / 'year1_only' / 'all' explicitly", then the key is
  ignored (no silent translation).

## 4. Output artefact changes

* **`02_dispatch/dispatch_hourly.xlsx`** — sheet count changes from N
  to N (one per operating year, **no Year-0 sheet**). The first sheet's
  name is `project_start_year` (Year 1), the last is
  `project_start_year + N - 1`.
* **`03_results.xlsx`** — sheet `cashflow_yearly` now has N+1 rows
  (Year 0 = CAPEX-only at `project_start_year - 1`; Years 1..N at
  `project_start_year .. project_start_year + N - 1`). The
  `cashflow_quarterly` and `cashflow_monthly` sheets are unchanged in
  schema (Years 1..N only).
* **`03_results.xlsx → financial_kpis`** — adds `capex_year`,
  `lcoe_eur_per_mwh`, `lcos_eur_per_mwh`, `pv_capacity_factor`,
  `bess_lifetime_cycles`, and the five Year-1 revenue-breakdown
  metrics.
* **`03_results.xlsx → rolling_horizon_compare_mc`** (new sheet) —
  produced when `uncertainty_compare_sources = TRUE`. Same columns as
  `rolling_horizon_mc` plus a `source_set` column with values `dam`
  / `pv` / `load` / `all`.
* **`04_financial_plots/`** — adds:
    * `revenue_stack_yearly_<start>-<end>.pdf`
    * `lifetime_cycles_<start>-<end>.pdf`
    * `lcoe_lcos_summary.pdf`
    * `rolling_horizon_foresight_gap_comparison.pdf` (new, when
      compare-sources is on).
* **`05_energy_plots/<calendar_year>/`** — adds new plots in
  merchant mode (`*_dispatch.pdf`, `*_soc.pdf`, `*_revenue.pdf`) at
  every active (daily / monthly / yearly) resolution. The vnb-mode
  plots are unchanged.
* **`05_energy_plots/<calendar_year>/daily/`** — now follows the
  same `none|year1_only|all` policy as monthly/yearly; selecting
  `all` over a 25-year horizon produces ~9 000 files (run-start
  WARNING).
* **Plot titles** — every financial plot's title changes from
  `... — <start>-<end>` (operating window only) to
  `... — <op_start>-<op_end> (CAPEX in <capex_year>)`.

## 5. The five-line "what changed" diff for v0.5 users

1. **Workbook is breaking**: regenerate it with `python
   scripts/build_input_xlsx.py` — the `# system` group splits into
   `# system_sizing` + `# bess_operation`, the `# optimization` group
   is removed, and `# uncertainty` is added.
2. **Year 0 = CAPEX year, not Year 1's twin**: a 20-year run with
   `project_start_year = 2026` puts CAPEX in 2025 and operations in
   2026..2045.
3. **`pv_nameplate_kwp = 0` / `bess_power_kw = 0` now mean "absent"**,
   not "infer" — three asset modes are first-class (PV-only,
   BESS-only, hybrid). Setting both to zero is an error.
4. **Rolling-horizon uncertainty is workbook-driven**, with an
   optional 4-source comparison
   (`uncertainty_compare_sources = TRUE`) that runs DAM-only,
   PV-only, Load-only, and All-combined ensembles.
5. **New financial KPIs and plots**: LCOE, LCOS, PV capacity factor,
   lifetime cycles, revenue-stack-by-year, lifecycle-cycles bar,
   LCOE/LCOS summary card; merchant mode also gets dedicated
   dispatch / SOC / revenue plots so its energy-plot folder is no
   longer empty.
