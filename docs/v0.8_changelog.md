# PV+BESS optimizer — v0.8 changelog

This file is the contract for the v0.8 feature pack. It enumerates
every file touched, every public-API signature change, every workbook
key added/renamed/removed, every breaking change to run outputs, and
the five-line "what changed" diff a v0.7 user will see.

The phases below are sequenced exactly as implemented:

* **Phase 1** — workbook split into seven themed sheets (breaking).
* **Phase 2** — BESS spec rationalisation; `e_cap` pinned to
  `bess_capacity_kwh` (no longer a decision variable).
* **Phase 3** — hour-of-day curtailment-cap profile (optional monthly
  axis).
* **Phase 4** — economics restructure: DEVEX, unavailability_pct,
  aggregator_fee_pct_revenue; `capex_licenses_eur_per_kw` dropped.
* **Phase 5** — IRR tornado redesign (dumbbell layout).
* **Phase 6** — LCOE/LCOS panel redesign (single comparison panel
  against Lazard 2024 benchmark bands).
* **Phase 7** — economic-model and uncertainty round-trip acceptance
  tests.
* **Phase 8** — documentation, README, version bump 0.7.0 → 0.8.0.

## 3. Workbook keys

### Added

* **`project` sheet**: `unavailability_pct` (default 1.0 %).
* **`pv` sheet** (new): `pv_nameplate_kwp`,
  `specific_production_kwh_per_kwp`, `pv_degradation_year1_pct`,
  `pv_degradation_annual_pct`, `capex_pv_eur_per_kw`,
  `devex_pv_eur_per_kw` (NEW; default 60 EUR/kWp),
  `opex_pv_eur_per_kwp`.
* **`bess` sheet** (new): `bess_power_kw`, `bess_capacity_kwh`,
  efficiency / SOC / cycle keys, `capex_bess_eur_per_kw`,
  `devex_bess_eur_per_kw` (NEW; default 30 EUR/kW),
  `opex_bess_eur_per_kw`, `bess_replacement_year`,
  `bess_replacement_cost_pct`, `bess_degradation_annual_pct`.
* **`economics` sheet** (new): `discount_rate_pct`,
  `opex_inflation_pct`, `revenue_inflation_pct`,
  `aggregator_fee_pct_revenue` (NEW; default 10.0 %), the five
  `sensitivity_*` keys.
* **`simulation` sheet** (new): the 11 `uncertainty_*` keys plus the
  three `plot_*_scope` flags.
* **`curtailment_profile` sheet** (new): hour-of-day cap profile
  (24×1 or 24×12).

### Removed

* `capex_licenses_eur_per_kw` — replaced by `devex_pv_eur_per_kw` /
  `devex_bess_eur_per_kw`.
* `battery_hours`, `p_charge_max_kw`, `p_dis_max_kw` — replaced by
  the symmetric `bess_power_kw` / `bess_capacity_kwh` pair.
* `curtailment_pct` (scalar on `# regulatory`) — replaced by the
  `curtailment_profile` sheet.

### Loader behaviour for legacy v0.7 workbooks

A v0.7-style two-sheet workbook (with `project` + `economic` sheets)
still loads through a backward-compat path: a single migration
WARNING points the user at
`python scripts/build_input_xlsx.py`.  The dropped keys
(`capex_licenses_eur_per_kw`, `battery_hours`, `p_charge_max_kw`,
`p_dis_max_kw`) emit a per-key WARNING and are ignored.  The legacy
v0.5 `# optimization` keys still warn unchanged.

## 5. The five-line "what changed" diff for v0.7 users

1. **Workbook is breaking**: regenerate it with `python
   scripts/build_input_xlsx.py` — the two-sheet `project + economic`
   layout becomes seven themed sheets.
2. **BESS surface rationalised**: `battery_hours`, `p_charge_max_kw`,
   `p_dis_max_kw` are gone; use `bess_power_kw` (symmetric) and
   `bess_capacity_kwh` (pinned).  `e_cap` is no longer optimised.
3. **Three new economic levers**: `devex_pv_eur_per_kw` and
   `devex_bess_eur_per_kw` replace `capex_licenses_eur_per_kw`;
   `unavailability_pct` (default 1 %) post-solve derate;
   `aggregator_fee_pct_revenue` (default 10 %) on gross revenue.
4. **Hour-of-day curtailment cap** lives on the new
   `curtailment_profile` sheet (24×1 or 24×12).  Missing sheet ⇒ flat
   27 %.
5. **Plot redesigns**: IRR tornado switches to a dumbbell layout and
   LCOE/LCOS becomes a single comparison panel against the Lazard
   2024 benchmark bands.

# v0.8.1 — patch round (LCOE audit, retail/DAM split, plot polish)

A patch release on top of v0.8.0 covering six related fixes.

## 1. Revenue inflation split

`revenue_inflation_pct` is replaced by two streams matching industry
practice (Lazard / Aurora / Gridcog do NOT index wholesale exports
to CPI):

* `retail_inflation_pct` (default 2.0 %) — load-coverage / PPA /
  VNB tariff. Applied to `profit_load_from_pv_eur` and
  `profit_load_from_bess_eur`.
* `dam_inflation_pct` (default 0.0 %) — wholesale DAM. Applied to
  `profit_export_from_pv_eur`, `profit_export_from_bess_eur`, and
  `expense_charge_bess_grid_eur`.

Legacy workbooks that still carry `revenue_inflation_pct` get a
WARNING and the value is mapped to `retail_inflation_pct` — no
breakage. The `cashflow_yearly` table now exposes per-stream
`revenue_retail_eur` and `revenue_dam_eur` columns; their sum
equals `revenue_eur` exactly (the aggregator fee is split in
proportion to gross stream contribution).

## 2. LCOE / LCOS computation audit

LCOE per IEA / IRENA / NREL ATB is PV-only. The previous
implementation summed PV and BESS CAPEX, DEVEX and OPEX into the
numerator, which inflated the LCOE for hybrid projects. The
corrected numerator uses `capex_pv_eur_per_kw * pv_kwp` + PV DEVEX
+ PV-only OPEX (escalated by `opex_inflation_pct`). LCOS was
already BESS-only and is unchanged.

Lazard 2024 EUR-equivalent benchmark bands (LCOE+ v17, LCOS v9 at
~1.08 EUR/USD):

* LCOE PV utility-scale: 30-85 EUR/MWh (was 30-50 — too narrow).
* LCOS 4-hour Li-ion: 157-274 EUR/MWh (was 100-250 — low edge
  too aggressive).

Four `benchmark_lcoe_*` / `benchmark_lcos_*` workbook keys are now
exposed for per-project overrides. A single INFO log line records
the computed LCOE / LCOS values next to the bands in `run_log.txt`.

## 3. NPV waterfall plot

`plot_npv_waterfall` now morphologically mirrors
`plot_yearly_cashflow_bars`: five-entry legend (Revenue, OPEX,
DEVEX, CAPEX, Net cash-flow) plus the Cumulative NPV line. In-axis
DEVEX / CAPEX captions next to the Year-0 bar are removed — the
legend conveys them. Y-axis margins padded so the topmost bar no
longer touches the spine.

## 4. cumulative_cashflow vs payback_visualization dedup

`plot_cumulative_cashflow` no longer draws payback verticals
(those duplicated `plot_payback`). The `plot_payback` output file
is renamed from `payback_visualization.pdf` to
`cumulative_cashflow_with_payback_<start>-<end>.pdf` for naming
parity.

## 5. LCOS annotation placement fix

In `lcoe_lcos_summary.pdf` the LCOS summary annotation used to land
inside the grey Lazard benchmark band when the project value was
to the left of the band (typical undershoot scenario). The summary
now lifts above the project bar in that case; overlapping projects
keep the historical right-of-bar placement.

## 6. Net revenue / Real-EUR net line colour

In `revenue_stack_yearly` the Net revenue line and dashed Real-EUR
net line used the dark purple `FINANCIAL_COLORS["discounted"]`,
which got swallowed by the dark blue BESS-export stack. Both lines
now use a dedicated `FINANCIAL_COLORS["net_revenue_line"]` set to
magenta (#E91E63), high-contrast over both PV light blue and BESS
dark blue stacks.

# v0.8.2 — plot-styling round (centralized labels, IEEE polish)

A second patch on top of v0.8.1 covering five plot-styling fixes.

## 1. Centralized FINANCIAL_LABELS / FINANCIAL_LEGEND_ORDER

Mirrors the energy plots' `ALL_LABELS` / `COLORS` / `LEGEND_ORDER`
pattern.  `pvbess_opt.config` now exposes:

* `FINANCIAL_LABELS` — the 17 canonical labels drawn on any
  financial / lifecycle plot (line series, bar / stack components,
  revenue-stack subcomponents).
* `FINANCIAL_LABEL_TO_COLOR_KEY` — binding from each label to its
  `FINANCIAL_COLORS` key.  Multiple labels may share a key
  (e.g. "Net cash-flow" and "Cumulative cash-flow" both read off
  `"net"`).
* `FINANCIAL_LEGEND_ORDER` — left-to-right / top-to-bottom order
  used by every financial-plot legend.  Lines first, then positive
  bars, then negative bars.
* `financial_color(label)` — raises `ValueError` if the label is
  not canonical, so typos surface at plot time.
* `apply_financial_legend(ax)` — drop-in replacement for
  `ax.legend(...)` that reorders entries to match
  `FINANCIAL_LEGEND_ORDER`.  Year-annotated payback labels
  ("Simple payback: 5.0 yr") are matched via a prefix path so the
  rendered text still carries the value while the canonical order
  is preserved.
* `UNCERTAINTY_SOURCE_COLORS` — per-source-set palette for the
  rolling-horizon comparison plots, moved out of
  `plotting/uncertainty.py` so the full palette lives in one file.

Every `ax.bar(...)` / `ax.plot(...)` in
`plotting/financial.py`, `plotting/lifecycle.py`, and
`plotting/uncertainty.py` routes its colour through
`financial_color(...)` or a `FINANCIAL_COLORS[...]` key — no hex
literal remains in the plotting modules.

## 2. LCOE / LCOS Lazard caption placement

The "Lazard: 30–85 EUR/MWh" / "Lazard: 157–274 EUR/MWh" captions
moved from the centre of the grey band (y=0) to below it (y=-0.50,
va="top").  Both rows place the caption identically.  Row y-limits
widened from `(-0.6, 0.6)` to `(-0.9, 0.6)` (or `(-0.9, 0.85)` when
the project summary lives above the bar) to accommodate.  Eliminates
the LCOE-row collision where the project bar overlapped the Lazard
band.

## 3. NPV total annotation inside the axes frame

`plot_npv_waterfall` switched the final-year "NPV = €X.XM"
annotation from a data-coordinate offset (which spilled past the
right spine on 20-year horizons) to an axes-coordinate placement at
`(0.98, 0.96)` with `transform=ax.transAxes`.  Mirrors the
total-cycles annotation in `plot_lifetime_cycles`.  The
`set_xlim(xmin, xmax + 1.5)` data-axis expansion is no longer
required and was removed.

## 4. IEEE-friendly emphasis colour for Net revenue lines

`FINANCIAL_COLORS["net_revenue_line"]` swapped from magenta
`#E91E63` (too loud for IEEE publication style) to near-black
`#212121` (Material grey 900).  In `plot_revenue_stack_yearly`:

* The solid `Net revenue` marker line gains `markeredgecolor="white"`
  + `markeredgewidth=0.6` so it reads cleanly over both light-blue
  and dark-blue stack segments.
* The dashed `Real-EUR net (deflated)` companion line keeps the
  same near-black hue and distinguishes itself by linestyle only
  (no markers — standard convention for "derived" series).

A new `perfect_foresight` colour key (same charcoal) aliases this
emphasis hue for the rolling-horizon distribution plot's
benchmark line.

## 5. Centralized labels in plot_monthly_cashflow_year1

`plot_monthly_cashflow_year1` now uses the canonical
`label="Net cash-flow"` (was `"Net"`) and routes its three series
colours through `financial_color(...)`.  The hardcoded
`("Revenue", "OPEX", "Net")` legend-reorder block is replaced by
`apply_financial_legend(ax, max_rows=2)`.

## Test contract

* `tests/test_financial_label_consistency.py` — new module
  covering: every label has a colour, every label is in the legend

  monthly cashflow legend uses `"Net cash-flow"`, the helper
  reorders correctly, and year-annotated payback labels are

* `tests/test_lcoe_lcos_redesign.py` — adds
  `test_lazard_caption_below_bar` and
  `test_lcoe_row_caption_below_even_when_bar_overlaps`.
* `tests/test_npv_waterfall_redesign.py` — adds
  `test_npv_total_annotation_inside_frame`; the legacy source-grep
  test for in-axis DEVEX/CAPEX captions migrates to a
  text-artist inspection on the rendered figure.
