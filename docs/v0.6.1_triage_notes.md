### Single failing test

`tests/test_uncertainty_config.py::test_foresight_gap_comparison_plot`

```
TypeError: boxplot() got an unexpected keyword argument 'orientation'
```

Stack:
- `tests/test_uncertainty_config.py:175` calls
  `plot_foresight_gap_comparison(df, tmp_path / "boxes.pdf")`.
- `pvbess_opt/plotting/uncertainty.py:144` then calls
  `ax.boxplot(data, orientation="horizontal", patch_artist=True,
  widths=0.6, tick_labels=sources)`.

Immediate cause: matplotlib's `Axes.boxplot` gained an `orientation=`
kwarg only in **matplotlib 3.10.0** (Dec 2024), where it began
deprecating the older `vert=` kwarg. matplotlib 3.10 also dropped
support for Python 3.9, so a 3.9 runner is forced onto
matplotlib 3.9.x — which does **not** have `orientation` yet.

Confirmed by inspecting the installed `boxplot` signature per venv:

- Py 3.9 + matplotlib 3.9.4 → has `vert`, **no** `orientation`.
- Py 3.11 + matplotlib 3.10.9 → has both
  (`vert` raises `PendingDeprecationWarning` under `-W error`).
- Py 3.12 + matplotlib 3.10.9 → has both.

The breaking call was introduced in v0.6 Phase 4 (workbook-driven
rolling-horizon uncertainty + 4-source compare, commit `a164167`) —
`plot_foresight_gap_comparison` is new in that phase.
